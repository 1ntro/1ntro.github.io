XSS总结

发表于 2018-11-29 | 分类于 XSS攻击

同源策略

浏览器的同源策略，限制了来自不同源的“document”或脚本，对当前“document”读取或设置某些属性
影响源的因素：host,子域名,端口,协议

a.com通过以下代码：
<script src=http://b.com/b.js ><s/script>
加载了b.com上的b.js，但是b.js是运行在a.com页面中的，对于当前打开的页面(a.com)来说，b.js的源就是a.com

XSS概述

XSS即跨站脚本，主要是利用网站本身设计不严谨，导致执行用户提交的恶意js脚本，对网站造成危害。XSS是web应用中最常见的漏洞，也是web中最主流的攻击方式

XSS分类

  • 反射型
    只是简单的把用户的输入“反射”给浏览器，配合攻击者的点击实现，通常也叫非持久型
  • 存储型
    存储型XSS也叫持久型XSS，payload经过存储，每次访问页面都将触发XSS，这种XSS具有很强的稳定性
  • DOM型
    这种类型的XSS并不是按照“数据是否保存在服务器端”来划分，从效果来看也可以说是反射型XSS，它通过修改页面的DOM节点形成XSS

XSS攻击进阶

  1. Cookie劫持，直接登陆用户账户
  2. 伪造请求，用JS模拟GET、POST请求操作用户的浏览器
  3. 钓鱼（用假页面或弹窗伪造真实应用，欺骗用户账号密码）
  4. 获取用户系统信息，识别用户的浏览器信息，用户安装的软件以及真实的IP等信息
  5. XSS Wrom，这是XSS的一种终极利用方式，破坏力和影响力巨大

XSS防御

  1. 给关键的Cookie设置HttpOnly属性
  2. 检查输入的逻辑，禁用不必要的标签
  3. 禁止外部调用js
  4. 对输出进行编码，黑白名单过滤<script>等关键标签以及一些事件
  5. 对常见的符号进行编码或过滤,例如PHP中的htmlspecialchars(),htmlentities()
  6. 最后还可以限制输入的长度强制截断
