vulnhub-DeRPnStiNK靶机渗透

发表于 2018-12-07 | 分类于 渗透

环境

靶机下载地址：百度网盘
网络配置为NAT模式

flag1

首先nmap -sP 192.168.1.1/24扫描一下靶机地址

确认192.168.1.131为靶机地址，访问http://192.168.1.131

查看源码得flag1

flag2

扫描目录

尝试访问http://192.168.1.131/weblog，发现无法跳转http://derpnstink.local/weblog/，想起源码中的webnotes/info.txt提示：<– @stinky, make sure to update your hosts file with local dns so the new derpnstink blog can be reached before it goes live –>

将192.168.1.131 derpnstink.local加入hosts文件再访问，Linux的hosts文件在/etc目录下

成功访问后发现是一个WordPress博客地址，WordPress的后台地址为http://域名/wp-admin或http://域名/wp-login.php，于是登陆http://derpnstink.local/weblog/wp-admin，尝试账号密码admin/admin成功进入后台

WPScan是一款WordPress的专用扫描器，使用方法看这里利用wpscan扫描WordPress漏洞wpscan --enumerate u u[10-20] ap at --url http://192.168.1.131/weblog/

发现几个漏洞和账号密码，然后利用MSF登陆拿shell

用cat命令查看wp-config.php，找到数据库账号密码

然后登陆phpmyadminhttp://derpnstink.local/php/phpmyadmin，在wp_posts中拿到flag2

flag3

在wp_users中拿到hash密文

先辨别hash密文的类型

利用john破解，得到wedgie57

用账号密码登陆FTP，得到两个文件：ssh的登陆密钥key.txt、和mrderp用户的对话derpissues.txt

把密钥复制到本地登陆sshssh -i key.txt stinky@192.168.1.131，（这里key.txt的权限设为700，如果为777会报错，别问我怎么知道的）

成功登陆后在Desktop发现flag.txt得到第三个flag

flag4

在Documents目录下发现一个数据包，下载下来用wireshark分析

利用得到的账号密码切换到用户mrderp，在Desktop目录下发现helpdesk.log

执行sudo -l，查看权限

得知mrderp用户只能在特定目录下执行derpy*文件使用sudo，所以创建binaries目录，并在derpy.py中写入/bin/bash，sudo执行

在/root/Desktop/下拿到flag4

总结

第一次尝试渗透测试，踩了很多坑，也有很多收获。渗透的一般姿势1.信息收集 2.漏洞扫描 3.漏洞利用 4. 权限提升。对Kali中工具的使用一定要熟练，对一些基本的命令要熟悉。本次学习主要用到了NMAP、WPScan、MSF、WireShark等工具。emmmm，就这样吧
